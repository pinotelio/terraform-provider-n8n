# n8n_credential (Resource)

Manages an n8n credential. Credentials store authentication information for external services and APIs used by workflow nodes.

## Example Usage

```terraform
resource "n8n_credential" "example" {
  name = "My API Credential"
  type = "httpBasicAuth"

  data = jsonencode({
    user     = "myusername"
    password = "mypassword"
  })
}

# Example with Datadog API credential
resource "n8n_credential" "datadog" {
  name = "Datadog Account"
  type = "datadogApi"

  data = jsonencode({
    apiKey = "xoxb-your-datadog-api-key"
    appKey = "xoxb-your-datadog-app-key"
  })
}

# Example with Datadog API credential using n8n environment variables
resource "n8n_credential" "datadog_env" {
  name = "Datadog Account"
  type = "datadogApi"

  data = jsonencode({
    apiKey = "{{ \\$.env.DATADOG_API_KEY }}"
    appKey = "{{ \\$.env.DATADOG_APP_KEY }}"
  })
}
```

- `name` (String) Name of the credential

### Read-Only

- `id` (String) Credential identifier

## Import

Credentials can be imported using their ID:

```shell
terraform import n8n_credential.example 1
```

## Notes

- The `data` field is marked as sensitive and will not be displayed in logs
- Credential types must match the types supported by your n8n instance
- When a credential is deleted, it is permanently removed from n8n
- Ensure no workflows are using a credential before deleting it
